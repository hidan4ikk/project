# G4ip-решатель с ML-эвристикой выбора хода

Решатель для интуиционистской логики высказываний (исчисление G4ip / LJT
Дикхоффа) и эвристика на градиентном бустинге, ранжирующая применимые ходы,
чтобы сокращать перебор.

## Пайплайн

```bash
python tests.py       # тесты корректности решателя, признаков, генератора
python solver.py      # датасет точек ветвления -> branch_dataset.jsonl
python ml.py          # обучение эвристики -> heuristic.pkl (+ диагностика)
python benchmark.py   # ГЛАВНАЯ метрика: поиск с эвристикой и без
```

Зависимости: Python 3.10+, `numpy`, `scikit-learn` (нужны только для
`ml.py` и `benchmark.py`; сам решатель и тесты — на чистой стандартной
библиотеке).

## Файлы

* `solver.py` — формулы, парсер, признаки, поиск вывода (`prove`) с
  поддержкой эвристики (`chooser`), статистики (`SearchStats`) и бюджета,
  генератор случайных формул, генерация датасета. Также интерактивный режим:
  `python solver.py --interactive`.
* `ml.py` — модель ранжирования ходов: по паре (состояние, ход) предсказывает,
  приведёт ли ход к доказательству; использует и proof-, и dead_end-строки.
  Обучение и офлайн-диагностика только на точках с реальным выбором
  (`len(choices) > 1`), метрики усредняются по нескольким сплитам по формулам
  и печатаются рядом с baseline'ами.
* `benchmark.py` — итоговая оценка: доля задач, решённых в бюджет, число
  посещённых секвентов и откатов — с эвристикой, без неё и с заведомо плохим
  (обратным) порядком как контролем чувствительности. Два набора задач:
  случайные формулы и заведомо доказуемые нетривиальные (`~~F` для
  классических тавтологий `F`, по теореме Гливенко).
* `tests.py` — известные теоремы/не-теоремы, кросс-проверка решателя
  таблицами истинности (корректность + полнота через теорему Гливенко),
  свойства признаков и генератора, схема журнала.
