# IPL Prover — прувер интуиционистской логики с ML-выбором правил

Автоматический решатель для **интуиционистской логики высказываний** (IPL — Intuitionistic Propositional Logic) на основе секвенциального исчисления, дополненный модулем машинного обучения: модель учится предсказывать, какое правило вывода стоит применить в текущем состоянии поиска доказательства.

## Идея

Поиск доказательства — это перебор с бэктрекингом. В каждом состоянии (секвенте `Γ ⊢ G`) допустимо несколько правил вывода, и неудачный выбор уводит в тупик, заставляя решатель откатываться. Порядок, в котором пробуются правила, напрямую влияет на скорость поиска.

Во время работы решатель логирует каждую точку ветвления: признаки состояния, список доступных ходов, сделанный выбор и его исход (`proof` / `dead_end`). На собранных данных обучается классификатор, который по признакам состояния предсказывает правило, ведущее к доказательству. Его можно использовать как эвристику: пробовать перспективные ходы первыми и тем самым сокращать перебор.

## Пайплайн

```
solver.py                            ml.py
random_formula + prove    ───▶    branch_dataset.jsonl    ───▶    rule_model.pkl
(генерация и разметка)            (лог ветвлений)                 rule_encoder.pkl
```

1. **Сбор данных.** `solver.py` генерирует 500 случайных формул (глубиной до 5), пытается доказать каждую и пишет лог всех ветвлений в `branch_dataset.jsonl`.
2. **Обучение.** `ml.py` отбирает успешные выборы (`result == "proof"`), обучает `GradientBoostingClassifier` и сохраняет модель (`rule_model.pkl`) вместе с кодировщиком меток правил (`rule_encoder.pkl`). Разбиение train/test выполняется через `GroupShuffleSplit` с группировкой по исходной формуле — состояния одной и той же формулы не попадают одновременно в обучение и тест, что исключает утечку данных.
3. **Интеграция (следующий шаг).** Подключить модель к `prove()`: сортировать ходы из `collect_moves` по предсказанной перспективности вместо фиксированного порядка.

## Исчисление

Решатель реализует правила в духе contraction-free исчисления **LJT** (Р. Дайкхофф): левое правило для импликации разбито на случаи по виду посылки, благодаря чему поиск завершается без специальной проверки зацикливаний.

| Правило | Условие | Действие |
|---|---|---|
| `Ax` | цель — переменная, и она есть в Γ | лист доказательства |
| `L_bot` | `false ∈ Γ` | лист доказательства |
| `R->` | цель `A -> B` | добавить `A` в Γ, доказывать `B` |
| `R&` | цель `A & B` | две подцели: `A` и `B` |
| `R\|1` / `R\|2` | цель `A \| B` | доказывать левый или правый дизъюнкт |
| `L&` | `A & B ∈ Γ` | заменить на `A, B` |
| `L\|` | `A \| B ∈ Γ` | разбор случаев: две ветки |
| `L->atom` | `p -> C ∈ Γ` и `p ∈ Γ` | добавить `C` в Γ |
| `L->&` | `(A & B) -> C ∈ Γ` | каррировать: `A -> (B -> C)` |
| `L->\|` | `(A \| B) -> C ∈ Γ` | заменить на `A -> C, B -> C` |
| `L->->` | `(A -> B) -> C ∈ Γ` | ветки `Γ, B -> C ⊢ A -> B` и `Γ, C ⊢ G` |

## Синтаксис формул

Связки по убыванию приоритета: `~` (отрицание), `&`, `|`, `->` (правоассоциативная). Переменные — идентификаторы (`p`, `q`, `x1`, …), константа лжи — `false`. Отрицание раскрывается стандартно для интуиционизма: `~A ≡ A -> false`.

Доказуемые примеры:

```
p -> p
(p & q) -> (q & p)
p -> ~~p
(p -> q) -> (~q -> ~p)
```

Недоказуемые (классически верны, но интуиционистски — нет):

```
p | ~p                  # исключённое третье
~~p -> p                # снятие двойного отрицания
((p -> q) -> p) -> p    # закон Пирса
```

## Признаки состояния

Каждая точка ветвления описывается вектором из 9 признаков секвента:

| Признак | Что это |
|---|---|
| `gamma_size` | число формул в контексте Γ |
| `goal_type` | главная связка цели (`var` / `and` / `or` / `imp` / `bot`) |
| `formula_size` | суммарный размер всех формул секвента |
| `max_depth` | максимальная глубина формулы в секвенте |
| `num_vars`, `num_and`, `num_or`, `num_imp`, `num_bot` | счётчики вхождений переменных и связок |

Формат записи в `branch_dataset.jsonl` (одна строка — одна точка ветвления):

```json
{"state": {"gamma_size": 1, "goal_type": "imp", "...": "..."},
 "choices": ["R->"], "choice": "R->", "choice_id": 0,
 "premises": 1, "result": "proof", "selected": "R->",
 "formula": "(p->(q->p))"}
```

## Установка и запуск

Требуется Python 3.8+.

```bash
pip install numpy scikit-learn
```

Сгенерировать датасет и обучить модель:

```bash
python solver.py    # создаст branch_dataset.jsonl
python ml.py        # выведет accuracy и сохранит rule_model.pkl, rule_encoder.pkl
```

Интерактивный режим прувера:

```bash
python -c "import solver; solver.main()"
```

```
Введите формулу (пустая строка — выход): p -> p
Доказуема. Дерево вывода:
|- (p->p)   [R->]
  p |- p   [Ax]
```
